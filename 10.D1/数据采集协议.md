### 二、传感器配置

| 参数 | 设定值 | 说明 |
|------|--------|------|
| 加速度计量程 | ±8g | 覆盖步行(±1.5g)、跑步(±3g)、跌倒(±6g) |
| 陀螺仪量程 | ±2000°/s | 覆盖快速旋转与跌倒冲击 |
| 磁力计量程 | 默认 | HMC5883L 默认配置 |
| 采样率 | **50 Hz** | 人体运动主要频率 ≤20 Hz，50 Hz 满足奈奎斯特采样定理 |
| 佩戴部位 | **腰部**（腰带/臂带固定） | 靠近身体质心，最能反映整体运动状态 |
| 佩戴方向 | 传感器 Y 轴指向身体前进方向，Z 轴指向地面 | 统一坐标系便于后续分析 |
| 固定方式 | 弹性绑带 + 硬质外壳 | 避免传感器相对身体晃动引入噪声 |

### 三、活动类别定义

| 类别编号 | 活动名称 | 描述 | 持续时间 |
|----------|----------|------|----------|
| 0 | 静坐 | 坐在椅子上，保持静止，可自然微动 | ≥60 s |
| 1 | 站立 | 自然站立，可轻微重心转移 | ≥60 s |
| 2 | 平地步行 | 正常步速（~1.5 m/s）平地行走 | ≥60 s |
| 3 | 跑步 | 中等速度慢跑（~3 m/s） | ≥60 s |
| 4 | 上楼 | 正常步速上楼梯（台阶高度 ~15 cm） | ≥60 s |
| 5 | 下楼 | 正常步速下楼梯（台阶高度 ~15 cm） | ≥60 s |
| 6 | 跌倒（进阶） | 在软垫上模拟向前/侧向跌倒后静止 | ≥10 s（含跌倒瞬间+静止） |

### 四、采集方案

| 参数 | 设定值 |
|------|--------|
| 每类活动每被试采集次数 | ≥ 3 次 |
| 每次持续时间 | ≥ 60 s（跌倒除外：≥10 s） |
| 采集间休息 | ≥ 30 s（避免疲劳导致动作变形） |
| 采集顺序 | 先静态后动态：静坐 → 站立 → 步行 → 上楼 → 下楼 → 跑步 → 跌倒 |
| 过渡段处理 | 每次采集**前后各裁掉 2 s**，去除启动/停止的过渡动作 |

#### 采集环境
- 步行/跑步：室外平直道路或操场跑道
- 上楼/下楼：教学楼楼梯（≥2 层连续台阶）
- 静坐/站立：室内
- 跌倒：体操垫/软垫上

---

### 五、标签协议

| 要素 | 方案 |
|------|------|
| 打标方式 | 采集前口头倒计时"3-2-1-开始"，同时按下标记按钮 / 串口发送标记指令 |
| 标签记录 | 在数据文件中新增 `label` 列，采集过程中由记录员实时填写 |
| 过渡标注 | 活动切换前后各裁 2 s（在预处理阶段执行） |
| 标签校验 | 采集结束后回放数据，人工核对标签与活动是否对齐 |

### 六、数据格式

#### 6.1 CSV 落盘格式

每被试每类活动一个文件，命名规范：`{SubjectID}_{ActivityName}_{Trial}.csv`

```
timestamp, ax, ay, az, gx, gy, gz, mx, my, mz, label, subject_id
```

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| timestamp | int64 | ms | 采集时间戳（相对起点） |
| ax, ay, az | float32 | m/s² | 三轴加速度（标定后） |
| gx, gy, gz | float32 | °/s | 三轴角速度（标定后） |
| mx, my, mz | float32 | μT | 三轴磁场（标定后） |
| label | int | — | 活动类别编号（0–6） |
| subject_id | str | — | 被试编号（S01, S02, ...） |

#### 6.2 目录结构

```
data/
├── raw/                          # 标定前的原始数据
│   ├── S01/
│   │   ├── S01_sit_01.csv
│   │   ├── S01_sit_02.csv
│   │   ├── S01_sit_03.csv
│   │   ├── S01_stand_01.csv
│   │   └── ...
│   ├── S02/
│   └── S03/
├── calibrated/                   # 标定后的数据
│   ├── S01/
│   ├── S02/
│   └── S03/
└── calib_params.json             # 标定参数
```

### 七、数据划分策略（防泄漏核心）

| 规则 | 说明 |
|------|------|
| **划分粒度** | 按 `subject_id` 划分，非按窗口随机划分 |
| **硬性禁止** | 同一被试的数据**绝对不能**同时出现在训练集和测试集中 |
| **推荐方案** | 留一被试法（LOSO）——轮番以 1 名被试为测试集，其余为训练集 |
| **原因** | 按窗口随机划分会导致同一被试的相邻窗口分散在训练/测试集，高估泛化能力（经典数据泄漏陷阱） |

```
示例（3 被试 LOSO）：
  Fold 1: 训练=[S01, S02]  测试=[S03]
  Fold 2: 训练=[S01, S03]  测试=[S02]
  Fold 3: 训练=[S02, S03]  测试=[S01]
```

### 八、采集前 Checklist

- [ ] 传感器已按 D2 标定，`calib_params.json` 已生成
- [ ] ESP32 固件已烧录，采样率验证为 50 Hz
- [ ] SD 卡 / 串口日志记录正常
- [ ] 绑带牢固，传感器不晃动
- [ ] 被试已签署知情同意（课程内可口头确认）
- [ ] 采集环境安全（尤其跑步与跌倒测试）
- [ ] 记录表已打印/电子版就绪
- [ ] 备用电/充电宝充足
