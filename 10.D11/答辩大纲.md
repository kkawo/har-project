# 答辩大纲 — HAR 可穿戴人体活动识别系统

> 专周实训 D11 答辩 | 5分钟陈述 + 3分钟提问

---

## 1. 项目简介 (30秒)

**一句话**：从零到一构建了一套可穿戴人体活动识别系统——PCB设计、传感器标定、数据采集、特征工程、模型训练、端侧部署全链路打通。

**7 种活动**：静坐 / 站立 / 步行 / 跑步 / 上楼 / 下楼 / 跌倒

**硬件**：ESP32-S3 + MPU6050 (6轴IMU)，腰部佩戴，50Hz采样

---

## 2. 工程全链路 (2分钟) — 重点

```
PCB设计(KiCad) → 嘉立创打样 → 手工焊接
    ↓
传感器标定(六位置法) → 去除系统误差
    ↓
固件开发(MicroPython) → 离线采集 / 实时推理
    ↓
2人真实数据采集 → 各7活动×3次 = 21 trial = 363窗口
    ↓
预处理(标定/去重力/滤波/窗口化) → 128样本窗口(2.56s)
    ↓
特征提取(14时域统计量×6轴) = 84维特征
    ↓
逻辑回归(LogisticRegression) → Softmax 7分类
    ↓
导出模型参数 → ESP32端侧推理
```

---

## 3. 关键技术决策 (1分钟)

| 决策 | 选择 | 原因 |
|------|------|------|
| 模型 | 逻辑回归 | ESP32内存有限，588次乘法<2ms |
| 特征 | 84维手工时域 | 无FFT，纯加减乘除，MicroPython可算 |
| 传感器 | 6轴(无磁力计) | 磁力计受环境干扰大，实际贡献小 |
| 窗口 | 2.56s / 50%重叠 | 人体活动周期~1-2s, 128样本足够 |
| 部署 | MicroPython软浮点 | 不用C编译，迭代快 |

---

## 4. 端侧部署核心问题与解决 (1分钟)

| 问题 | 解决方案 |
|------|----------|
| 中位数/四分位数排序耗内存 | 用均值近似median, 1.349σ近似IQR |
| 过零率(ZCR)训练/推理不一致 | 固件ZCR减均值后再检测过零(scipy一致) |
| 模型参数如何导出 | 训练后格式化写入.py文件，ESP32直接import |
| 充电宝供电I2C卡死 | init_mpu建新SoftI2C + 3次重试 |
| 插座和电池传感器量程不同 | 统一±8g / ±2000°/s, 寄存器显式配置 |

---

## 5. 结果展示 (30秒)

- **训练精度**：100% (同人自用场景)
- **实时推理**：50Hz采样 → 每1.28s输出一次活动识别
- **LED指示**：7色NeoPixel实时显示当前活动
- **HTTP Dashboard**：手机浏览器访问ESP32 IP查看

---

## 6. 总结

**最大亮点**：不是调包跑个模型，而是从硬件到算法到部署的**全栈工程实践**。

**核心能力**：遇到问题→定位根因→工程化解决（I2C调试/特征对齐/内存优化/端侧部署）。

**未来方向**：多被试数据扩充、时序平滑(HMM)、1D-CNN端到端对比。
