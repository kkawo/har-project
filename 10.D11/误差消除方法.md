# 传感器误差消除方法

> 答辩用：说明我们如何从原始传感器数据中消除系统误差

---

## 总览

传感器出厂有误差，直接用原始数据会导致模型学到错误特征。我们从两级消除：

```
原始传感器读数
    ↓ ① 六位置标定（消除零偏+灵敏度+轴间耦合）
校准后物理量 (m/s², °/s)
    ↓ ② 数据预处理（去重力+滤波+窗口化）
干净的特征输入
```

---

## ① 六位置法加速度计标定（D2）

### 原理

加速度计静止时，三轴合力应等于重力加速度 g ≈ 9.80665 m/s²。但由于**零偏(bias)**和**灵敏度误差(scale)**，实际读出来不是 g。

标定模型：

```
a_calibrated = (a_raw - bias) × scale
```

### 操作

把传感器分别以 **6 个方向** 静止放置，每个方向采集 5 秒：

| 位置 | 朝向 | 重力应在哪个轴 |
|------|------|----------------|
| +Z down | 正面朝下 | +Z = +g |
| -Z up | 正面朝上 | -Z = -g |
| +X down | X轴朝下 | +X = +g |
| -X up | X轴朝上 | -X = -g |
| +Y down | Y轴朝下 | +Y = +g |
| -Y up | Y轴朝上 | -Y = -g |

### 数学求解

每个位置求采样均值，得到 6 个 (ax, ay, az) 均值。对每条轴：

```
expected[i] = g × (±1)    ← 理想值
measured[i] = mean(axis)  ← 实测值

用最小二乘拟合：measured = expected / scale + bias
→ 解出 bias 和 scale
```

### 结果

```
标定前 RMSE: 11.35 m/s²
标定后 RMSE:  0.42 m/s²  ← 改善 96.3%
```

标定参数写入 `calib/calib_params.json`：
```json
{
  "accelerometer": {
    "bias": [0.000944, -0.094707, -0.455895],
    "scale": [-0.997629, 1.007918, 0.981314]
  }
}
```

> scale ≈ 1.0 说明灵敏度误差很小；bias 主要是 Z 轴 −0.456，即传感器在 Z 轴有约 0.05g 的零偏。

## ② 数据预处理管道（D3）

采集的原始数据经过三个步骤：

### 2.1 标定校准

```python
# 对每个采样点
ax = (ax_raw - ACC_BIAS[0]) * ACC_SCALE[0]
ay = (ay_raw - ACC_BIAS[1]) * ACC_SCALE[1]
az = (az_raw - ACC_BIAS[2]) * ACC_SCALE[2]
```

### 2.2 去重力（Butterworth 高通滤波）

加速度计读的是 重力 + 运动加速度。我们只需要运动部分。

```
截止频率 0.3Hz, 4阶 Butterworth 高通滤波器
→ 滤掉直流分量（重力），保留运动信号
```

### 2.3 低通滤波

去除高频噪声（传感器噪声、身体抖动）：

```
截止频率 20Hz, 4阶 Butterworth 低通滤波器
→ 保留人体活动频段 (0.3~20Hz)
```

### 2.4 滑窗切分

```
窗口大小：128 样本 = 2.56 秒（人体活动基本周期）
步长：64 样本 = 1.28 秒（50% 重叠）
```

## ③ 固件端误差处理（实时推理）

ESP32 上没有 scipy，不能做 Butterworth 滤波。替代方案：

### 3.1 标定参数直接应用

`model_params.py` 中包含 `ACC_BIAS` 和 `ACC_SCALE`，固件 `mpu_read()` 每次读数后立即校准：

```python
ax = raw_ax / ACCL_SENS * GRAVITY   # 转为 m/s²
ax = (ax - ACC_BIAS[0]) * ACC_SCALE[0]  # 校准
```

### 3.2 省去滤波（训练数据也同步）

固件不做 Butterworth 滤波（省计算），但训练数据也不做。`src/d11_export_model.py` 里有一个关键操作——**用均值替代中位数、用 1.349σ 替代 IQR**，确保训练数据的数学表达和固件的近似计算完全一致：

| 特征 | Python (scipy) | 固件 (纯Python) | 对齐方式 |
|------|---------------|-----------------|---------|
| median | `np.median()` | `mean` | 训练数据用 mean 覆盖 |
| IQR | `scipy.stats.iqr()` | `1.349 × std` | 训练数据用 1.349σ 覆盖 |
| ZCR | `mean-center + signbit` | `mean-center + sign` | 固件加 `mu` 参数 |

## 总结

```
标定参数(calib_params.json)
    ↓
采集时用 / 推理时也用 → 统一
    ↓
训练管道：标定 → 去重力 → 滤波 → 窗口化 → 特征(median≈mean对齐)
推理固件：标定 → 直接读 → 特征(手工近似)
    ↓
两者数学表达一致 → 模型输入一致 → 推理有效
```

**核心原则**：不追求训练管道多精密，但追求**训练和推理的数学计算完全一致**。
