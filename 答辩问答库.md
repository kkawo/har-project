## 一、项目概述类

### Q1: 你们做的是什么项目？一句话概括。

用绑在腰部的 MPU6050 + HMC5883L 采集加速度/角速度/磁场数据，经过标定→预处理→特征提取→特征选择→分类器训练→LOSO评估，构建一个能自动区分 7 种日常活动（静坐/站立/步行/跑步/上楼/下楼/跌倒）的模式识别系统。

### Q2: 为什么要做这个？有什么实际应用？

- **健康监护**：老人跌倒检测、日常活动量追踪
- **运动分析**：运动员训练量化
- **智能家居**：根据用户活动自动调节环境
- **产业方向**：可穿戴设备核心算法（Apple Watch/小米手环都在做 HAR）

### Q3: 你们的系统和手机/手表自带的计步器有什么区别？

- 手机计步器只做**二分类**（走/不走），我们做 **7 类**精细识别
- 手机只用加速度计，我们加了**陀螺仪+磁力计**，能区分上楼/下楼
- 我们完成了**传感器标定**——手机直接用原始数据，我们用标定参数修正了系统误差
- 我们做了**代价敏感决策**——跌倒漏检代价×10，这是产品级安全设计

## 二、传感器标定类（D2 核心，被抽问概率高）

### Q4: 为什么要做传感器标定？不标定会怎样？

**未标定的原始数据带有系统误差**：
- 加速度计：静止时合加速度≠1g（零偏+刻度误差）
- 磁力计：3D 轨迹是偏心椭球（硬铁偏置+软铁形变）
- 陀螺仪：静态有零偏输出

这些误差会让后续所有特征"失真"，**识别率天花板被死死压住**。标定就是把"原始数据"变成"可信数据"。

### Q5: 加速度计六位置标定怎么做？数学原理是什么？

**采集**：把传感器静止摆放在 6 个姿态（±X、±Y、±Z 各朝上），每姿态采 5-10s 取均值。

**模型**：12 参数误差模型 `a_calibrated = M × (a_raw - b)`，M 是 3×3 刻度+非正交矩阵，b 是 3×1 零偏。

**约束**：静止时校正后合加速度模长应为 1g = 9.81 m/s²。

**求解**：以"模长=1g"构造方程组，最小二乘求解 12 参数。**最小二乘正是线性回归/线性判别的数学基石**。

### Q6: 磁力计椭球拟合怎么做？SVD 在其中起什么作用？

**现象**：未标定磁力计 3D 数据是偏心椭球。

**采集**：手持传感器空中画 8 字 + 多轴旋转，覆盖各方向。

**拟合**：直接最小二乘 (DLS) 拟合一般二次曲面 `ax²+by²+cz²+2fxy+2gxz+2hyz+2px+2qy+2rz+d=0`。

**SVD 的作用**：对拟合的二次型矩阵做特征分解，求出椭球中心（硬铁偏置）与形变矩阵（软铁），把椭球校正为球。**SVD 与 PCA/K-L 变换同根——这正是 Ch10 降维的核心工具**。

### Q7: Allan 方差是什么？有什么用？

长时间静置（≥30 min）采集陀螺仪数据，画 Allan 方差 log-log 曲线，读取：
- **角随机游走 (ARW)**：高频白噪声
- **零偏不稳定性 (BI)**：低频漂移

理解噪声的统计特性，衔接 Ch2/Ch3 的高斯假设。

### Q8: 你们标定的实际结果如何？

- 加速度计：RMSE 从 11.35 → 0.42 m/s²
- 磁力计：norm_std 从 7.44 → 4.78 uT（35.8% 改善），覆盖度偏低 (0.21)，需重做
- 陀螺仪：零偏 [-4.29, -1.21, 0.67] °/s

## 三、预处理与特征工程类

### Q9: 窗口参数为什么选 2.56s / 128 点 / 50% 重叠？

- **2.56s**：足够包含 1-2 个完整步态周期（步行 ~2 Hz），同时保证系统响应速度
- **128 点 = 2.56s × 50Hz**：FFT 友好的 2 的幂
- **50% 重叠**：HAR 文献标准，平衡样本量和时间分辨率

### Q10: 为什么要去重力？用什么方法？

加速度计测量的是**运动加速度+重力加速度**的叠加。重力是低频分量（~0 Hz），用 **0.3 Hz 高通 Butterworth 滤波**分离，保留 >0.5 Hz 的步态频段。

### Q11: 你们有哪些特征？为什么选这些？

**294 维特征** = 9 轴 × (14 时域 + 10 频域) + 3 组模长 × (14+10) + 12 跨轴/复合

- **时域**：均值/方差/RMS/峰峰值/偏度/峰度/过零率/SMA/IQR/自相关 → 运动强度和分布
- **频域**：主频/频谱质心/谱熵/频带能量比/带宽/平坦度/滚降点/谱峰数 → 步频与运动规律
- **关键特征**：合加速度 IQR（运动强度）、垂直/水平能量比（上下楼判别）、磁航向标准差（转向检测）

### Q12: 为什么说上楼 vs 下楼是最难区分的？

两者步态周期性相近，步频接近，仅**垂直加速度方向相反**（上推开 vs 下冲击），单一特征难以区分。

**解决思路**：
- 频谱质心差异（下楼高频能量更大）
- 垂直加速度偏度符号
- `acc_vertical_ratio`（垂直/水平能量比）——我们特征矩阵中排名第 7

## 四、特征选择与降维类

### Q13: Filter / Wrapper / Embedded 三类方法有什么区别？

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **Filter** | 用统计量（ANOVA F / 互信息）独立评分每个特征 | 快，不依赖分类器 | 不考虑特征组合效果 |
| **Wrapper** | 用分类器精度反馈搜索最优子集（SFS 逐个加） | 考虑特征交互 | 计算量大 |
| **Embedded** | 训练过程中自动选特征（L1 正则 / RF 重要性 / RFE） | 中等开销，效果通常最好 | 依赖模型假设 |

**我们的策略**：ANOVA Top-100 预筛 → SFS 精选 20 → RF MDI 验证

### Q14: PCA 和 LDA 有什么区别？什么时候用哪个？

| | PCA | LDA |
|------|-----|-----|
| **类型** | 无监督 | 有监督 |
| **目标** | 最大化方差 | 最大化类间/类内距离比 |
| **不需要** | 标签 | 需要标签 |
| **上限** | 无限制 | n_components ≤ n_classes - 1 |
| **使用场景** | 探索性分析、降维预处理 | 分类前的判别特征提取 |

**我们的结果**：PCA 25 维保留 95% 方差，LDA 最多到 6 维（7 类-1）。

### Q15: t-SNE 是用来做什么的？能用来做分类吗？

t-SNE 是**可视化工具**，把高维数据嵌入 2D 空间，保持相似样本靠近、不相似样本远离。

**不能用来做分类**——t-SNE 是非参数随机嵌入，没有 learned transformation，新样本无法映射。只用于"提前预判模型上限"——如果 t-SNE 图上不同颜色混在一起，说明特征空间本身难分。

## 五、分类器类

### Q16: 最小错误率贝叶斯和最小风险贝叶斯有什么区别？

- **最小错误率**：选后验概率最大的类 → argmax P(ω|x)
- **最小风险**：考虑不同错误代价不同 → argmin E[cost(α, ω)]

**跌倒案例**：
- 普通误分类 cost=1
- 跌倒漏检（真跌倒判非跌倒）cost=10

代价矩阵会让模型更"保守"地判跌倒——**整体准确率可能微降，但跌倒召回率显著提升**。

### Q17: 为什么 LDA 在你们的 demo 数据上只有 25.6%？

285 维特征 × 1698 样本 → 类内协方差矩阵接近奇异（样本数 < 特征数的 7 倍）。

高维小样本下 LDA 的 SVD 求解器在极端多重共线性下失效。

**解决方案**：先用 PCA 降到 25 维再输入 LDA。

### Q18: Bagging 和 Boosting 的区别？RF 和 GBDT 各适用什么场景？

| | Bagging (RF) | Boosting (GBDT) |
|------|-------------|----------------|
| **训练方式** | 并行训练多个树 | 串行，每棵树纠正前一棵的错误 |
| **偏差/方差** | 降方差 | 降偏差 |
| **过拟合风险** | 低 | 中等（需调 lr） |
| **适用** | 高维小样本，特征多 | 大数据集，有足够样本 |

**我们的结论**：demo 数据 RF 更稳（100% vs GBDT 100%，但 RF 无需调 lr），真实数据上预期 GBDT 在大样本时更优。

### Q19: SVM 的核函数是做什么的？什么时候用 RBF 不用线性？

**核函数 = 隐式高维映射**，把原始空间线性不可分的数据映射到高维空间变得线性可分。

- **线性核**：特征已足够多（如 285 维），数据大致线性可分 → 快
- **RBF 核**：需要非线性决策边界 → 更灵活，但需要调 C 和 gamma

## 六、模型评估类（D8 核心，被抽问概率高）

### Q20: 为什么要用 LOSO 而不是随机 k-fold？数据泄漏是怎么回事？

**随机 k-fold 的陷阱**：同一被试的相邻窗口（50% 重叠，间隔仅 1.28s）高度相关。如果这些窗口分散在训练/测试集，模型"见过"测试被试的几乎相同数据 → **泛化能力被严重高估**。

**LOSO**：每个被试的所有窗口要么全在训练集，要么全在测试集 → 真正测试"对没见过的人"的泛化能力。

**这是企业级红线**。任务书明确指出：按窗口随机划分导致数据泄漏 → **直接不合格**。

### Q21: McNemar 检验是干什么的？p < 0.05 说明什么？

检验两个模型 A 和 B 的**错误率差异是否统计显著**（而非随机波动）。

构建 2×2 列联表：同时正确/同时错误/A 对 B 错/A 错 B 对，卡方检验。

p < 0.05 表示"A 和 B 表现相同"这个零假设被拒绝 → A 和 B 有显著差异。

**当前 demo 数据**：所有模型都 100%，McNemar 无法区分。真实数据上才有意义。

### Q22: Bootstrap 置信区间怎么理解？

从原始预测结果中**有放回地重采样** 1000 次，每次计算准确率，取 2.5% 和 97.5% 分位数作为 95% CI。

含义：如果我们重复这个实验 100 次，真实的准确率有 95% 的概率落在这个区间内。

## 七、工程实践类

### Q23: Pipeline 如何防泄漏？具体代码怎么写的？

**关键原则**：标准化（StandardScaler）的 fit 只能在训练集上调用。

```python
# 正确做法 (sklearn Pipeline)
pipe = Pipeline([("scaler", StandardScaler()), ("clf", LinearSVC())])

# LOSO 每 fold:
for train_idx, test_idx in logo.split(X, y, groups):
    pipe.fit(X[train_idx], y[train_idx])      # fit scaler ONLY on train
    y_pred = pipe.predict(X[test_idx])         # transform test with train's params
```

**错误做法（泄漏）**：
```python
# ❌ 全量数据上标准化 → 测试集信息泄漏进训练
scaler = StandardScaler()
X_all = scaler.fit_transform(X)
```

### Q24: ESP32 上怎么实现实时推理？

1. 在 PC 上训练好模型（如 LinearSVM / 小 RandomForest）
2. 提取模型参数（系数、截距、树结构）
3. 用 Python/C 在 ESP32 上重写预测函数（不需要 sklearn）
4. 实时采集→滑窗→特征提取（C 实现，快速）→模型推理→输出活动标签

**挑战**：50Hz 采集 → 每 1.28s（50% 重叠）出一次结果，特征计算和推理必须在 <1.28s 完成。

### Q25: PCB 设计有哪些关键约束？

- HMC5883L 距 ESP32 主芯片/WiFi 天线 ≥15mm（避免电磁干扰）
- I²C 上拉 4.7kΩ×2、去耦 100nF×2 靠近芯片
- 2 层板，≤60×30mm，嘉立创 2 层板 5 片约 ¥5
- 留模块脚位降风险（不裸焊 QFN）

## 八、开放问题类（考察理解深度）

### Q26: 如果给你更多时间，你会怎么改进这个系统？

1. **数据层面**：增加被试（≥10 人）、增加佩戴位置（腕部/脚踝）
2. **特征层面**：加入小波变换时频特征、加入气压计（上下楼直接可区分）
3. **模型层面**：尝试 1D-CNN 端到端（自动学特征 vs 手工特征对比）
4. **工程层面**：模型量化→TF-Lite Micro→ESP32 实时推理，测功耗-时延-精度三角
5. **评估层面**：做消融实验（去掉某组特征看精度变化），做跨被试/跨位置域自适应

### Q27: 这个项目中你学到的最重要的东西是什么？

1. **数据质量决定上限**：标定把 RMSE 从 11 降到 0.4 → 这是所有算法的前提
2. **评估严谨性是区分"会调包"和"真懂算法"的分水岭**：LOSO vs 随机划分、McNemar 检验
3. **工程闭环思维**：从硬件到算法到评估，每个环节的决策都影响最终结果
4. **代价敏感设计**：跌倒漏检×10 → 算法设计要考虑真实场景的后果
