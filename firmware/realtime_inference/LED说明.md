# ESP32 实时推理 — NeoPixel LED 灯光说明

> 硬件：ESP32-S3 + NeoPixel RGB LED @ GPIO48  
> 固件：`firmware/realtime_inference/main.py`

---

## 一、系统状态灯

| 状态 | LED 表现 | 含义 |
|------|----------|------|
| 初始化中 | 🔴 红色快闪 ×10 | 传感器初始化失败，检查 I²C 接线 |
| 就绪 | ⚪ 白色慢闪 ×3 | 传感器 OK，等待佩戴并开始活动 |
| 采集中 | 无额外灯效 | 正常读取传感器，等待推理触发 |
| 传感器读错 | 无灯效（仅串口报错） | 偶发 I²C 错误，已自动恢复 |

---

## 二、活动识别灯（核心）

每 1.28 秒更新一次，颜色随识别结果实时变化。亮度与置信度成正比（置信度越高越亮）。

| 活动 | 颜色 |  RGB   | 视觉 | 备注 |
|------|------|--------|------|------|
| 静坐 sit | 🔵 蓝 | (0, 0, 255) | 稳定蓝色 | |
| 站立 stand | 🔷 青 | (0, 255, 255) | 稳定青色 | |
| 步行 walk | 🟢 绿 | (0, 255, 0) | 随步频微闪 | 最高置信度活动 |
| 跑步 run | 🟡 黄 | (255, 255, 0) | 明亮黄色 | |
| 上楼 upstairs | 🟠 橙 | (255, 165, 0) | 暖橙色 | |
| 下楼 downstairs | 🟣 紫 | (128, 0, 255) | 深紫色 | |
| 跌倒 fall | 🔴 红闪 | (255, 0, 0) | **快速闪烁 3 次** | 区别于其他活动！ |

## 四、使用说明

1. **上电** → 等待 3 次白灯闪烁（约 3 秒）→ 系统就绪
2. **佩戴** → 传感器固定在腰部，Y 轴指向前方
3. **开始活动** → LED 每 1.28 秒更新一次，显示当前识别结果
4. **跌倒检测** → 红色闪烁为跌倒警告，持续 3 个闪烁周期后恢复
5. **串口同步查看** → 连接串口（115200 baud）可看到详细置信度

## 五、故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 上电无灯 | 供电不足/焊接断路 | 检查 USB 供电，量 R3/D1 通路 |
| 红灯快闪后熄灭 | I²C 传感器未检测到 | 检查 SDA(IO17)/SCL(IO18) 接线 |
| 灯始终白色 | 模型参数未加载 | 确认 model_params.py 已上传 |
| 灯色不变 | 模型过拟合到某类 | 更换活动，观察是否切换 |
| 灯色频繁跳动 | 跨活动过渡或模型不确定 | 正常现象，置信度 <50% 时常见 |
