# HAR 项目快速启动

## 目录结构

```
firmware/offline_logger/main.py       → 离线采集固件（刷到 ESP32 采数据）
firmware/realtime_inference/main_wifi.py → 实时推理固件（刷到 ESP32 推理用）
firmware/realtime_inference/model_params.py → 模型参数（训练后自动生成）
dashboard/app.py                      → 实时 Dashboard（电脑跑）
dashboard/test_serial.py              → 串口测试工具
src/                                  → Python 训练管道
docs/                                 → 文档
10.D11/                               → 答辩材料
```

---

## 一、在线实时推理 + Dashboard

### 1. 刷固件到 ESP32

用 Thonny 打开 `firmware/realtime_inference/main_wifi.py`，**另存为 ESP32 上的 `main.py`**（自动运行）。

同时确保 `model_params.py` 也在 ESP32 上。

### 2. 启动 Dashboard

```
python dashboard/app.py
```

浏览器打开 `http://localhost:5000`

Dashboard 自动检测并连接 ESP32 串口。如果连不上：关掉 Thonny → 拔插 ESP32 USB → 刷新页面。

---

## 二、离线数据采集

### 1. 修改被试编号

打开 `firmware/offline_logger/main.py`，改第 44 行：
```python
SUBJECT = "S01"  # 或 "S02"
```

### 2. 刷固件 & 佩戴传感器

Thonny 打开 `main.py` → Run。传感器绑腰带（怎么绑都行，但采集和推理必须一致）。

### 3. 看灯光做动作

| 颜色 | 动作 |
|------|------|
| 蓝 | 静坐 |
| 青 | 站立 |
| 绿 | 步行 |
| 黄 | 跑步 |
| 橙 | 上楼 |
| 紫 | 下楼 |
| 红 | 跌倒 |
| 常亮 | 录制中 |
| 白闪 | trial 完成 |

### 4. 下载数据

采集完成后连电脑，Thonny 下载所有 CSV 文件到 `data/raw/real2/S01/`。

---

## 三、训练模型 & 导出

```bash
python src/preprocess.py          # 预处理 → data/windowed/
python src/features.py            # 特征提取 → data/features/
python src/d11_export_model.py    # 训练+导出 → firmware/realtime_inference/model_params.py
```

重新刷 `model_params.py` 到 ESP32 即可用新模型。

---

## 四、串口测试

如果 Dashboard 不显示数据，先确认串口能读到 ESP32 输出：

```
python dashboard/test_serial.py
```

应该看到 `★ DATA:{"id":0,"zh":"静坐",...}` 行。如果看不到：
1. 关掉所有 Thonny 窗口
2. ESP32 拔 USB → 等 3 秒 → 插回去
3. 确认 ESP32 上存有 `main.py`（独立运行时自动执行）
4. 重试

---

## 五、常见问题

| 问题 | 解决 |
|------|------|
| Dashboard 连不上 | 关 Thonny → 拔插 ESP32 → 重开 dashboard |
| 网页显示不动 | 确认串口指示灯是否亮，点"连接硬件"重试 |
| I²C 报错 ENODEV | 拔 USB 硬复位（不要软重启），重新 Run |
| 预测不准 | 确认 `model_params.py` 是最新的；确认传感器佩戴方向跟采集时一致 |
